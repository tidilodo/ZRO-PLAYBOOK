# Skill — Product Context Engineering

Versão: 1.0

---

# Objetivo

Product Context Engineering (PCE) é a metodologia oficial da empresa para criação de produtos digitais.

Seu objetivo é garantir que todo projeto seja construído sobre contexto sólido antes da implementação.

A metodologia reduz retrabalho, melhora a comunicação entre humanos e Inteligências Artificiais e cria produtos preparados para evoluir durante muitos anos.

---

# O Problema

Grande parte dos projetos começa pelo código.

Isso normalmente resulta em:

- arquitetura inconsistente;
- mudanças frequentes de direção;
- documentação incompleta;
- retrabalho;
- baixa escalabilidade.

A causa não é falta de capacidade técnica.

É falta de contexto.

---

# Nossa Tese

Não desenvolvemos software.

Construímos contexto.

O software é apenas uma consequência.

Quando o contexto está correto:

- a arquitetura melhora;
- o código melhora;
- a documentação melhora;
- as decisões tornam-se mais simples.

---

# Quando utilizar

Esta Skill deve ser aplicada em:

- novos produtos;
- novos SaaS;
- plataformas;
- sistemas internos;
- APIs;
- aplicativos;
- produtos baseados em IA.

Ela pode ser utilizada independentemente da tecnologia adotada.

---

# Fluxo da Metodologia

Todo projeto deve seguir obrigatoriamente esta sequência.

```
Discovery

↓

Vision

↓

Architecture

↓

Context

↓

Backlog

↓

Implementation

↓

Evolution
```

Cada etapa possui um documento próprio.

Nenhuma etapa deve ser ignorada.

# Etapas

## 01 — Discovery

Compreender o problema.

Identificar oportunidades.

Validar hipóteses.

---

## 02 — Vision

Definir propósito.

Público.

Transformação.

Objetivos.

---

## 03 — Architecture

Organizar módulos.

Responsabilidades.

Fluxos.

Limites.

---

## 04 — Context

Construir a memória permanente do projeto.

Documentação.

Glossário.

Decisões.

Regras.

---

## 05 — Backlog

Transformar documentação em pequenas entregas.

---

## 06 — Implementation

Executar apenas uma tarefa por vez.

Sempre preservando arquitetura.

---

## 07 — Evolution

Revisar.

Aprender.

Melhorar.

Documentar.

---

# Resultado Esperado

Ao final da metodologia o projeto deverá possuir:

- visão clara;
- arquitetura consistente;
- documentação completa;
- backlog organizado;
- contexto permanente;
- implementação incremental.
