# ZRO PLAYBOOK

> O sistema operacional de engenharia de produtos da ZRO.

---

# O que é o ZRO Playbook?

O ZRO Playbook é o conjunto oficial de princípios, metodologias, padrões e processos utilizados pela ZRO para projetar, desenvolver e evoluir produtos digitais.

Ele funciona como a fonte da verdade para qualquer projeto desenvolvido pela empresa.

Seu objetivo é garantir que humanos e Inteligências Artificiais trabalhem utilizando a mesma linguagem, os mesmos critérios e a mesma arquitetura de pensamento.

---

# Nossa Filosofia

Nós não começamos projetos escrevendo código.

Começamos construindo contexto.

Antes de qualquer implementação existir, deve existir clareza sobre:

- Qual problema será resolvido.
- Para quem o produto existe.
- Qual transformação será entregue.
- Como essa solução poderá evoluir ao longo do tempo.

O código é consequência.

O contexto é o ativo principal.

---

# Objetivos

O ZRO Playbook foi criado para:

- Padronizar o desenvolvimento de produtos.
- Melhorar a comunicação entre humanos e IA.
- Reduzir retrabalho.
- Preservar decisões arquiteturais.
- Criar produtos fáceis de evoluir.
- Documentar conhecimento da empresa.

# Componentes

## FOUNDATION

Define os princípios fundamentais da empresa.

Responde perguntas como:

- Como pensamos?
- Como tomamos decisões?
- O que nunca fazemos?

---

## Skills

Cada Skill representa uma metodologia reutilizável.

Exemplos:

- Product Context Engineering
- Product Discovery
- Software Architecture
- Database Design
- AI Integration
- Documentation

---

## Templates

Modelos utilizados para iniciar novos projetos.

Exemplos:

- Vision
- Manifesto
- Architecture
- Business
- Roadmap
- Database
- Technical Design

---

## Checklists

Listas de verificação para garantir qualidade antes de avançar para a próxima etapa.

---

## Prompts

Prompts reutilizáveis para utilização com Inteligências Artificiais.

Todos seguem os padrões definidos neste Playbook.

---

## Examples

Projetos de referência.

Utilizados como material de estudo e exemplo para novos produtos.

---

# Como utilizar

Sempre que um novo projeto for iniciado:

1. Criar um novo repositório.
2. Copiar os templates necessários.
3. Definir visão e arquitetura.
4. Aplicar as Skills relevantes.
5. Gerar a documentação.
6. Criar o backlog.
7. Somente então iniciar a implementação.

# Regras

- Nunca iniciar um projeto pelo código.
- Toda decisão importante deve ser documentada.
- A documentação é a fonte oficial da verdade.
- Todo projeto deve ser modular.
- IA auxilia na implementação, mas não substitui decisões estratégicas.
- Todo produto deve ser pensado para evoluir continuamente.

# Para Inteligências Artificiais

Antes de executar qualquer tarefa:

1. Leia este README.
2. Leia o FOUNDATION.md.
3. Leia as Skills relacionadas.
4. Consulte os templates e checklists.
5. Somente então execute a tarefa solicitada.

Em caso de conflito entre código e documentação, a documentação prevalece.
