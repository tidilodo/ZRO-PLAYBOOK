# Checklist: Novo Projeto

Execute este checklist antes de escrever qualquer linha de código.

---

## Documentação de Contexto

- [ ] Criar `docs/VISION.md` — problema, público, transformação, objetivo
- [ ] Criar `docs/MANIFESTO.md` — crenças, valores, princípios, limites
- [ ] Criar `docs/ARCHITECTURE.md` — camadas, módulos, fluxos
- [ ] Criar `docs/GLOSSARY.md` — termos oficiais sem ambiguidade
- [ ] Criar `docs/BUSINESS.md` — modelo de negócio, métricas, riscos
- [ ] Criar `docs/ROADMAP.md` — fases com escopo e critérios
- [ ] Criar `docs/TECH.md` — stack com justificativas
- [ ] Criar `docs/DATABASE.md` — entidades, campos, sensibilidade

## Contexto Vivo

- [ ] Criar `context/product.md` — fase ativa, escopo atual, fora do escopo
- [ ] Criar `context/stack.md` — estado da stack (proposta / aceita / em revisão)
- [ ] Criar `context/ai.md` — papel da IA, limites, fluxo

## Tarefas

- [ ] Quebrar Fase 1 em tarefas executáveis (máx 1 dia cada)
- [ ] Cada tarefa tem: objetivo, requisitos, critérios de aceite, dependências
- [ ] Sequência respeita dependências

## Validação Final

- [ ] Todos os documentos preenchidos e revisados?
- [ ] Escopo da Fase 1 está fechado?
- [ ] Stack foi formalmente decidida (não apenas proposta)?
- [ ] Glossário cobre todos os termos do projeto?

**Somente após todos os itens marcados: iniciar programação.**
